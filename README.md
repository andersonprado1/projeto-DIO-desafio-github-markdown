# Projeto de Colaboração e Markdown DIO
## 📕 O projeto
### Primeiro projeto da Formação de Github Certification pela DIO. após os primeiros cursos o desafio abaixo foi proposto como forma de prática das teorias apresentadas e primeiros comandos desenvolvidos 

1. Criar sua conta no Github:
   1. Essencial para alcançarmos nossos objetivos.
2. Criar um Repositório:
   1. Crie um novo repositório no GitHub, nomeando-o como "desafio-github-markdown" ou outro nome.
   2. Adicione o seu README.md ao criar o repositório.
   3. Utilize o Github.dev para editar seu projeto diretamente com o editor web do Github.
   4. Lembre-se: o repositório deve ser público.
3. Colaboração:
   1. Adicione um colaborador ao seu repositório (pode ser um colega de curso ou você mesmo com uma conta diferente).
   2. Ambos devem fazer um clone do repositório em suas máquinas locais.
   3. Realizem alterações, Commits, abra issues e pull requests.
4. Formatação com Markdown:
   1. Edite o arquivo "README.md" em seu repositório.
   2. Utilize a linguagem Markdown para formatar informações sobre o desafio, instruções e uma breve descrição do seu projeto.
